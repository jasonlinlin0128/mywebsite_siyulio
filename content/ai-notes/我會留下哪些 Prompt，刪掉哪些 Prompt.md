不是每一個用過一次的 Prompt，都值得留下來。

如果什麼都存，最後只會得到一個越來越難找、也越來越不想打開的 Prompt 倉庫。所以我現在會刻意篩掉很多東西，只保留真的會重複使用的版本。

## 我會留下來的 Prompt

### 1. 能反覆用在同一類工作上的

例如：

- 會議摘要
- 當下筆記整理成待辦事項
- 郵件改寫
- 專案文件整理
- 簡報摘要
- 主管版摘要

這些有一個共同點，就是情境固定、問題重複，而且每次都要花時間重做。這種 Prompt 最值得留下來。

### 2. 改過兩次以上，明顯變好用的

有些 Prompt 第一次寫得很粗，但後來因為真的有在用，會慢慢知道：

- 需要先定義角色
- 要不要限制輸出格式
- 哪些條件沒講清楚就容易跑掉

只要一個 Prompt 被你調整過兩三次，通常就代表它已經開始從「臨時想到的一句話」變成「有方法的工具」。

像我自己會特別留下來的，通常就是這種已經在工作中反覆出現的場景：

- 會議結束後快速出摘要
- 把錄音和零散筆記整理成待辦
- 把原本太硬或太亂的信件改寫得更清楚
- 把一包專案資料先整理成結構化文件
- 幫簡報先生出內容骨架

### 3. 能直接複製就上手的

我會保留的 Prompt，通常不需要想太久就能用。

打開後最好一眼就知道：

- 它是拿來做什麼的
- 哪些欄位要替換
- 產出會長什麼樣子

如果一個 Prompt 每次打開都還要重新理解，那它的複用價值就很低。

## 我會刪掉的 Prompt

### 1. 只用過一次、而且情境太特殊的

有些 Prompt 是為了某次很特定的需求而寫的，事情做完之後幾乎不可能再用到。這種我現在會直接刪掉，不讓它佔位置。

### 2. 看起來很厲害，但自己根本沒在用的

網路上很多 Prompt 模板看起來很完整，但如果你自己的工作根本不會用到，它只會變成收藏焦慮。

我現在比較不想累積「看起來很多」，而是想累積「真的能用」。

### 3. 需要太多前置說明才能用的

如果一個 Prompt 要先補很多背景、很多限制、很多上下文，才有辦法動起來，那它通常不適合放進日常工具庫。

這種 Prompt 不是沒有價值，而是比較像專案型腳本，不像平常可以快速重複使用的模板。

## 我現在的判斷標準很簡單

我通常會用這三題過濾：

1. 我下週還會再用到嗎
2. 這個情境是不是會反覆出現
3. 我下次打開時能不能直接複製使用

只要有兩題答不出來，我大多就不留。

## Prompt 庫不是越大越好，而是越清楚越好

對我來說，真正有用的 Prompt 庫不需要很大。

它只要做到三件事就夠了：

- 找得到
- 用得上
- 用完能持續微調

如果能維持這三點，Prompt 才會真的變成個人工作流的一部分，而不是另外一個看起來很多、實際上很少打開的資料夾。
